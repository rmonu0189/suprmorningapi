API reference

JSON under /v1. Writable routes expect Content-Type: application/json. Admin mutations (create/update/delete for CMS and catalog) require an access JWT with role: "admin". Set Base URL in the sidebar; example cURL commands use it below.

Token flow & overview

Passwordless sign-in with one flow: send an OTP to any phone (new or existing). Verify the code; if no user exists for that phone, the server creates one (optional full_name and email on the verify request), then returns the same token shape as for returning users.

By default OTPs are logged to storage/logs/otp.log — replace with SMS in production (OtpNotifier).

Access JWT — short-lived. Send as Authorization: Bearer <access_token>. Payload includes sub (user id), role (e.g. user or admin), and typ: "access".

Admin-only APIs — Endpoints that create, update, or delete CMS or catalog data (currently POST, PUT, and DELETE on /v1/pages and /v1/brands) require the access JWT’s role claim to be exactly "admin". Set users.role = 'admin' in MySQL, then sign in again (or refresh) so the new token includes that role. Authenticated non-admins receive 403 with { "error": "Forbidden" }. Missing or invalid tokens receive 401.

Refresh token — opaque hex string, stored server-side as a hash. Use with POST /v1/auth/refresh and POST /v1/auth/logout. Each successful refresh rotates the refresh token; persist the new one.

Environment

  • JWT_ACCESS_TTL_SECONDS, JWT_REFRESH_TTL_SECONDS, JWT_SECRET, JWT_ISSUER, JWT_AUDIENCE
  • OTP_LENGTH, OTP_TTL_SECONDS, OTP_MAX_ATTEMPTS, OTP_LOG_TO_FILE, OTP_RETURN_CODE_IN_RESPONSE (dev only)
  • OTP_ENABLE_FIXED_TEST_OTP, OTP_TEST_CODE, OTP_TEST_PHONES — when enabled, listed phones (comma-separated, any format) receive a fixed code for local testing; must match OTP_LENGTH. Off by default in .env.example.

POST /v1/auth/otp/send

Creates or replaces the OTP challenge for a phone. Rate-limited per IP and per phone.

Body

  • phone (required) — 10–15 digits after normalization.
  • device_label, user_agent (optional).

Responses

  • 200 — { "ok": true, "expires_in": … }; dev may include debug_otp_code.
  • 403 — phone registered but account disabled.
  • 422 — invalid phone.
  • 429 — rate limit.
Example request body 
{ "phone": "+1 555 123 4567" }
Sample response — 200 OK 
{
  "ok": true,
  "expires_in": 600
}

If OTP_RETURN_CODE_IN_RESPONSE=true in .env (dev only): same object plus "debug_otp_code": "123456".

Sample error — 422 (invalid phone) 
{
  "error": "Invalid phone number",
  "errors": {
    "phone": "Provide 10–15 digits (country code included if applicable)."
  }
}
Sample error — 403 (account disabled) 
{ "error": "Account is disabled" }
Sample error — 429 (rate limit) 
{
  "error": "Too many OTP requests",
  "retry_after": 3542
}
curl -sS -X POST "/v1/auth/otp/send" \
  -H "Content-Type: application/json" \
  -d '{"phone":"+15551234567"}'

POST /v1/auth/otp/verify

Validates OTP and returns tokens. For new phones, full_name and email are optional; existing phones ignore profile fields. Lockout after repeated failures (phone + IP).

Body

  • phone, code (required).
  • full_name — optional when the phone has no account yet.
  • email — optional, new accounts only.
  • device_label, user_agent — optional.

Responses

  • 200 — user, access_token, refresh_token, expires_in.
  • 401 / 403 / 404 / 409 / 422 / 429 — see overview for semantics.
Example body — existing user 
{ "phone": "+1 555 123 4567", "code": "123456" }
Example body — new user (minimal) 
{ "phone": "+1 555 987 6543", "code": "123456" }
Example body — new user with profile 
{
  "phone": "+1 555 987 6543",
  "code": "123456",
  "full_name": "Ada Lovelace",
  "email": "ada@example.com"
}
Sample response — 200 OK 
{
  "user": {
    "id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
    "phone": "15551234567",
    "email": null,
    "full_name": null,
    "role": "user",
    "is_active": true,
    "created_at": "2026-03-29 10:00:00"
  },
  "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJzdWIiOiJhMWIyYzNkNC1lNWY2LTc4OTAtYWJjZC1lZjEyMzQ1Njc4OTAiLCJyb2xlIjoidXNlciIsInR5cCI6ImFjY2VzcyIsImlzcyI6InN1cHJtb3JuaW5nLWFwaSIsImF1ZCI6InN1cHJtb3JuaW5nLWNsaWVudHMiLCJpYXQiOjE3MTAwMDAwMDAsImV4cCI6MTcxMDAwMDkwMH0.SIGNATURE",

  "expires_in": 900
}

access_token is a JWT; refresh_token is a long opaque hex string — values above are shortened/illustrative.

Sample error — 401 (bad or expired OTP) 
{ "error": "Invalid or expired code" }
Sample error — 422 (validation) 
{
  "error": "Invalid code",
  "errors": { "code": "Enter the 6-digit code." }
}
Sample error — 429 (lockout) 
{
  "error": "Too many failed attempts",
  "retry_after": 842
}
curl -sS -X POST "/v1/auth/otp/verify" \
  -H "Content-Type: application/json" \
  -d '{"phone":"+15551234567","code":"123456"}'

POST /v1/auth/refresh

New access JWT and a new refresh token; previous refresh token is revoked.

Body

refresh_token (required). Optional device_label, user_agent.

Errors

401 invalid/expired refresh; 403 disabled user.

Sample response — 200 OK 
{
  "user": {
    "id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
    "phone": "919876543210",
    "email": "user@example.com",
    "full_name": "Riya Sharma",
    "role": "user",
    "is_active": true,
    "created_at": "2026-01-15 08:30:00"
  },
  "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "refresh_token": "NEW_OPAQUE_REFRESH_HEX_STRING_STORE_THIS",
  "expires_in": 900
}

Always save the new refresh_token; the old one no longer works.

Sample error — 401 
{ "error": "Invalid or expired refresh token" }
Sample error — 403 
{ "error": "Account is disabled" }
curl -sS -X POST "/v1/auth/refresh" \
  -H "Content-Type: application/json" \
  -d '{"refresh_token":"YOUR_REFRESH_TOKEN"}'

POST /v1/auth/logout

Revokes the given refresh token for this device/session. Other devices unchanged.

Example request body 
{ "refresh_token": "YOUR_CURRENT_REFRESH_TOKEN" }
Sample response — 200 OK 
{ "ok": true }
Sample error — 422 (missing token) 
{
  "error": "refresh_token is required",
  "errors": { "refresh_token": "Required." }
}
curl -sS -X POST "/v1/auth/logout" \
  -H "Content-Type: application/json" \
  -d '{"refresh_token":"YOUR_REFRESH_TOKEN"}'

GET /v1/pages

Public — no Authorization header. Load screen content for the app.

Query

  • Omit query → { "pages": [ … ] } — all rows, ordered by page_name, card_index, id.
  • ?page_name=home — only cards for that page name.
  • ?id=<uuid> — single row { "page": { … } }.

Row shape

id (uuid), card_index (int), page_name, type, content (decoded JSON object/array).

Errors

  • 404 — when ?id= is set but no row exists.
  • 422 — invalid id (not a UUID).
Sample response — 200 OK (list, e.g. ?page_name=home) 
{
  "pages": [
    {
      "id": "11111111-2222-3333-4444-555555555555",
      "card_index": 0,
      "page_name": "home",
      "type": "hero_banner",
      "content": {
        "title": "Good morning",
        "subtitle": "Daily needs delivered",
        "imageUrl": "https://cdn.example.com/banner.jpg"
      }
    },
    {
      "id": "aaaaaaaa-bbbb-cccc-dddd-eeeeeeeeeeee",
      "card_index": 1,
      "page_name": "home",
      "type": "product_row",
      "content": {
        "title": "Top picks",
        "productIds": ["p1", "p2", "p3"]
      }
    }
  ]
}
Sample response — 200 OK (single row, ?id=<uuid>) 
{
  "page": {
    "id": "11111111-2222-3333-4444-555555555555",

POST /v1/pages

Admin only — Authorization: Bearer <access_token> with JWT claim role: "admin" (set users.role in the DB, then sign in again).

Body

  • page_name (required), type (required), content (required JSON object or array).
  • card_index (optional, default 0).

Responses

  • 201 — { "page": { … } }.
  • 401 / 403 — missing token or not admin.
  • 422 — validation.
Example request body 
{
  "page_name": "home",
  "type": "hero_banner",
  "card_index": 0,
  "content": { "title": "Good morning", "subtitle": "Daily needs" }
}
Sample response — 201 Created 
{
  "page": {
    "id": "11111111-2222-3333-4444-555555555555",
    "card_index": 0,
    "page_name": "home",
    "type": "hero_banner",
    "content": {
      "title": "Good morning",
      "subtitle": "Daily needs"
    }
  }
}
Sample error — 403 (not admin) 
{ "error": "Forbidden" }
Sample error — 422 (validation) 
{
  "error": "Invalid type",
  "errors": { "type": "Required." }
}
curl -sS -X POST "/v1/pages" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_ADMIN_ACCESS_TOKEN" \
  -d '{"page_name":"home","type":"hero_banner","card_index":0,"content":{"title":"Hello"}}'

PUT /v1/pages

Admin only — same auth as POST /v1/pages. Partial update: send id plus any fields to change.

Body

  • id (required, uuid).
  • At least one of: page_name, type, content, card_index.

Responses

  • 200 — { "page": { … } }.
  • 404 — unknown id.
  • 401 / 403 — auth / not admin.
  • 422 — validation.
Example request body 
{ "id": "11111111-2222-3333-4444-555555555555", "content": { "title": "Updated" } }
Sample response — 200 OK 
{
  "page": {
    "id": "11111111-2222-3333-4444-555555555555",
    "card_index": 0,
    "page_name": "home",
    "type": "hero_banner",
    "content": {
      "title": "Updated"
    }
  }
}
Sample error — 404 
{ "error": "Not Found" }
curl -sS -X PUT "/v1/pages" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_ADMIN_ACCESS_TOKEN" \
  -d '{"id":"PAGE_UUID","content":{"title":"Updated"}}'

DELETE /v1/pages

Admin only — remove one row by id (query string or JSON body).

Parameters

  • ?id=<uuid> or JSON body { "id": "…" }.

Responses

  • 200 — { "ok": true }.
  • 404 — row not found.
  • 422 — missing/invalid id.
  • 401 / 403 — auth / not admin.
Sample response — 200 OK 
{ "ok": true }
Sample error — 404 
{ "error": "Not Found" }
Sample error — 422 (invalid/missing id) 
{
  "error": "Invalid id",
  "errors": { "id": "Provide a valid UUID via ?id= or JSON body." }
}
curl -sS -X DELETE "/v1/pages?id=PAGE_UUID" \
  -H "Authorization: Bearer YOUR_ADMIN_ACCESS_TOKEN"

DELETE /v1/brands

Admin only — remove one brand by id (?id= or JSON body). Fails with 409 if any products row still references this brand.

Parameters

  • ?id=<uuid> or JSON body { "id": "…" }.

Responses

  • 200 — { "ok": true }.
  • 404 — brand not found.
  • 409 — brand has products; remove or reassign them first.
  • 422 — missing/invalid id.
  • 401 / 403 — auth / not admin.
Sample response — 200 OK 
{ "ok": true }
Sample error — 409 (brand still has products) 
{
  "error": "Conflict",
  "errors": {
    "brand": "Cannot delete a brand that still has products. Remove or reassign products first."
  }
}
curl -sS -X DELETE "/v1/brands?id=BRAND_UUID" \
  -H "Authorization: Bearer YOUR_ADMIN_ACCESS_TOKEN"

CORS

Preflight OPTIONS returns 204. Methods and headers are set in SecurityHeaders; allowlist via CORS_ALLOW_ORIGINS in .env.

Database schema

MySQL definitions: database/schema.sql — tables users (includes role, default user), phone_otp_challenges, refresh_tokens, pages (id, card_index, page_name, type, content JSON), and catalog tables brands, products, variants, inventory. Optional upgrades: database/migrations/.
